# AskHeyStupid 🤖

AskHeyStupid is a serverless AI-powered web application built on AWS to demonstrate secure architecture, API design, and controlled LLM integration using Amazon Bedrock.

The application delivers intentionally humorous responses while showcasing real-world cloud engineering and **security operations practices**, including monitoring, threat detection, and remediation.

---

## 🚀 Overview

AskHeyStupid is a cloud-native application designed to simulate a production-style workload while integrating AI capabilities. Users submit questions through a web interface and receive short, intentionally absurd responses generated by Amazon Bedrock (Claude).

While the output is humorous, the underlying architecture emphasizes **security, observability, and risk reduction**.

---

## 🏗️ Architecture

Browser  
→ CloudFront (HTTPS, CDN)  
→ AWS WAF (rate limiting & abuse protection)  
→ S3 (Static Hosting - private origin)  
→ API Gateway (REST API)  
→ Lambda (Backend Logic)  
→ Amazon Bedrock (Claude)

## 🔐 Security Decisions

- **Origin Access Control (OAC)**  
  Restricted direct access to the S3 bucket, ensuring content is only served through CloudFront.

- **HTTPS Enforcement**  
  All traffic is encrypted in transit using CloudFront with enforced HTTPS.

- **Least-Privilege IAM Roles**  
  Scoped permissions to allow only required actions between Lambda, Bedrock, and supporting services.

- **Controlled API Exposure**  
  API Gateway acts as the single entry point to backend services, preventing direct access to Lambda.

- **AWS WAF Protection**  
  Implemented WAF with rate-based rules and managed protections to mitigate abuse, bot traffic, and excessive requests.

- **CORS Configuration**  
  Explicitly configured to allow only required methods and headers, reducing unintended exposure.

- **Reduced Attack Surface**  
  Eliminated unnecessary services (e.g., Bedrock Agents) to simplify architecture and minimize risk.

## 🛡️ Security Monitoring & Remediation

- **AWS Security Hub Integration**  
  Enabled Security Hub to aggregate findings across services and continuously monitor security posture.

- **AWS Config Enforcement**  
  Used AWS Config to track resource configurations and identify non-compliant settings.

- **Findings Remediation**  
  - Resolved all **critical and high severity findings**
  - Remediated approximately **50% of medium severity findings**
  - Applied configuration changes and IAM adjustments to reduce exposure

- **Continuous Improvement Approach**  
  Security posture is treated as an ongoing process, with iterative improvements based on findings and risk prioritization.

## 🧠 Design Decisions

- **S3 + CloudFront vs. Amplify**  
  Chose a direct S3 and CloudFront architecture over Amplify Hosting to explicitly demonstrate core AWS building blocks and security controls. While Amplify would have provided a faster, fully managed deployment, this approach offered greater visibility into configuration, including OAC, HTTPS enforcement, and origin behavior.

- **Controlled LLM Output (Amazon Bedrock – Claude)**  
  Integrated Amazon Bedrock (Claude) with intentional prompt design to control response style and behavior. Because the application purposefully diverges from the model’s default training (factual, relevant, and context-aware responses), prompts were engineered to bias output toward humor and irrelevance. This included reinforcing tone directives and increasing temperature to encourage more creative responses.

- **Direct Model Invocation vs. Bedrock Agents**  
  Initially explored using Bedrock Agents; however, testing showed that direct model invocation provided better control over output behavior and reduced architectural complexity. This decision improved predictability, simplified the request flow, and reduced the overall operational and security surface area.

## ⚙️ Challenges & Solutions

- **CORS Preflight Failures (403 Errors)**  
  Resolved by aligning API Gateway methods, headers, and Lambda response structure.

- **API Routing Issues via CloudFront**  
  Fixed incorrect origin routing where API requests were being sent to S3 instead of API Gateway.

- **IAM Permission Gaps**  
  Debugged and corrected missing permissions between services to ensure proper execution flow.

- **Security Findings Remediation**  
  Interpreted Security Hub findings and applied targeted fixes to improve overall security posture.

- **LLM Output Control**  
  Overcame challenges in constraining model responses through prompt engineering and configuration tuning.
